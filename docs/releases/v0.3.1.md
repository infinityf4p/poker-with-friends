# Poker with Friends v0.3.1

Released: 2026-07-13

## Highlights

- All player-facing copy now sounds like a casual game between friends. Login, lobby, invitations, ready states, live-table prompts, and empty states no longer lead with policy, audit, or administrative-limit explanations.
- Joining a table is shorter: signed-in players can enter directly without an extra disclaimer checkbox.
- The table header replaces the old generic “记录” button with a responsive “牌谱” control, a dedicated book icon, a compact mobile treatment, and clearer hover, press, focus, and reduced-motion behavior.
- Hand history receives a new replay summary, hand counter, illustrated empty state, and more cohesive desktop dialog and mobile bottom-sheet styling.
- Existing persisted room messages are translated to the friendlier copy at render time, so rooms created before this release do not keep showing outdated wording.
- Server origin checks, configuration validation, concurrent room joins, error redaction, PWA fallback handling, and reconnect behavior include the hardening fixes already merged on `main` after v0.3.0.

## Test coverage

- Formatting, type checks, production builds, 30 engine tests, 31 server tests, and 16 web tests pass.
- The PostgreSQL-backed HTTP and Socket.IO suite again completes a real three-player hand through all four streets and verifies concurrent room joins never exceed six seats.
- Desktop and 375 px mobile browser checks cover the table header, responsive layout, hand-history dialog, and persisted-message compatibility without horizontal overflow.

## Upgrade notes

This release has no database migration. Deploy the new application image over the existing PostgreSQL data and verify `/health/ready`, the build SHA, login, room reconnect, WebSocket actions, and hand history after the switch.

Rollback to v0.3.0 does not require a database rollback.
