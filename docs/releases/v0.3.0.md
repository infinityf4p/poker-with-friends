# Poker with Friends v0.3.0

Released: 2026-07-12

## Highlights

- Account creation now shows the accepted username, display-name, and password rules before submission and keeps validation or duplicate-account errors inside the dialog.
- Duplicate usernames return a stable `409 USERNAME_TAKEN` response; unexpected server failures no longer expose SQL text, password hashes, or internal exception details.
- Hand history is rebuilt as a responsive, per-hand accordion with winners, pot and refund breakdowns, public cards, and natural-language actions grouped by street.
- Mobile account management uses readable cards instead of a horizontally scrolling desktop table, while the table, positions, action controls, and dialogs receive larger touch targets and clearer status feedback.
- Modal dialogs now include focus containment and restoration, Escape handling, scroll locking, accessible names, and a layer above the mobile action dock.
- Mobile betting always exposes every legal action, including a direct All-in control when no normal bet or raise is legal.
- Static asset misses return `404` instead of the application shell, and the PWA only caches successful responses with a matching script or stylesheet content type.
- The UI publishes its application version in `data-app-version` to make deployed asset verification deterministic.

## Test coverage

- A new PostgreSQL-backed Socket.IO E2E suite drives three independent users through account creation, first-login password changes, seating, unanimous hand confirmation, all four betting streets, settlement, history, button rotation, heads-up positions, stale commands, and idempotent retries.
- CI now runs that real-table E2E suite against PostgreSQL 18 in addition to formatting, type checks, unit and property tests, builds, container checks, secret scanning, and CodeQL.
- Account-contract tests cover readable validation errors and duplicate-account response redaction.

## Upgrade notes

This release has no new database migration. Deploy the new application image, keep the existing PostgreSQL data, and verify `/health/ready`, login, account creation, room reconnect, WebSocket actions, and hand history before completing the release switch.

The PWA cache namespace changes from its first legacy namespace to the second; the new service worker removes the old cache after activation. Rollback to v0.2.0 does not require a database rollback.

## Product boundary

Poker with Friends continues to use non-redeemable entertainment points. This release adds no payment, cash-out, rake, escrow, or real-money settlement feature.
