# Poker with Friends v0.4.0

## Highlights

- The product is now named **Poker with Friends** across the application shell, PWA metadata, API identity, documentation, package workspace, Docker defaults, and operational templates.
- The canonical technical slug is `poker-with-friends`, and workspace packages now use the `@poker-with-friends/*` scope.
- The PWA cache moves to a new brand-specific namespace so returning clients receive the renamed application shell instead of a stale title or wordmark.
- The existing color palette, icon, gameplay, accounts, rooms, hand state, and database schema remain unchanged.

## Verification

- Brand regression coverage checks the PWA full and short names.
- Formatting, type checks, unit and property tests, production builds, Compose validation, and a repository-wide legacy-name scan must pass before release.
- Production verification covers the page title, visible wordmark, manifest name, build SHA, HTTPS health endpoint, and WebSocket application startup.

## Database and rollback

This release does not add a database migration. Rolling back to v0.3.3 does not require a database rollback, but clients will display the previous product identity again.
