# Poker with Friends 文案规范

## 语气

- 清晰：优先说明当前状态或可执行动作。
- 克制：不使用宣传口号、拟人表达或重复说明。
- 友好：使用自然中文，不暴露内部技术术语。

## 统一术语

| 使用       | 不使用               |
| ---------- | -------------------- |
| 牌桌       | 房间、朋友桌、这桌   |
| 管理员     | 开桌人、组局小助手   |
| 牌谱       | 记录、回看、牌局回放 |
| 筹码       | 积分                 |
| 下一手暂离 | 下手休息             |
| 准备下一手 | 人齐再开、我准备好了 |

## 规则

- 按钮使用动词或“动词＋对象”，不使用句子。
- 常驻界面只显示状态、数据与操作，不显示规则解释、操作教学或阵容变化叙述。
- 状态由阶段、行动人、座位、筹码和准备人数等结构化数据生成，不在牌桌状态中保存自由文案。
- 输入错误、操作失败和不可执行原因只在实际触发后显示，并保留完整信息。
- 错误提示不出现令牌、快照、序列等内部实现词汇。
- 首页之外不使用宣传性文案；品牌名称中的 `Friends` 不受此限制。
- 不使用输入帮助、提示段落或安全说明作为常驻占位内容。

## 示例

| 场景         | 文案                           |
| ------------ | ------------------------------ |
| 等待开局     | 等待玩家准备                   |
| 已确认       | 已准备                         |
| 状态过期错误 | 牌桌状态已更新，请重试         |
| 金额错误     | 请输入最小值至最大值之间的整数 |
| 加载失败     | 牌谱加载失败                   |
| 争议状态     | 结果存在异议，本手已暂停       |
